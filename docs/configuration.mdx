# Configuration

Most platform-level behavior is controlled in `docusaurus.config.js`, including branding, URLs, locales, and navigation. This file should be treated as release-critical because incorrect values here can break routes or cause deployment issues.

For content operations, use versioning and locale folders as your source of truth. Current content is edited in `docs/`, released snapshots are stored under `versioned_docs/`, and translated docs live under `i18n/<locale>/docusaurus-plugin-content-docs/current/`.

To keep releases stable, teams typically freeze docs at release time with `npm run docusaurus docs:version <version>`, then continue writing in current docs for upcoming changes. This pattern gives readers a reliable historical reference while keeping active documentation flexible.

## Core Paths

- Site config: `docusaurus.config.js`
- Current docs: `docs/`
- Version snapshots: `versioned_docs/`
- Version menu source: `versions.json`
- Translated docs: `i18n/fr/docusaurus-plugin-content-docs/current/`
- Theme and brand tokens: `src/css/custom.css`
