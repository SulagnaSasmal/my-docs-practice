# Troubleshooting

Most documentation issues in this project fall into one of three categories: build failures, deployment mismatches, or versioning confusion. A quick local build is the fastest way to isolate whether the problem is content-related or pipeline-related.

If CI fails but local preview works, compare your branch against `main` for config differences in `docusaurus.config.js`, `versions.json`, and `.github/workflows/deploy.yml`. In most cases, the error is caused by a broken relative link, an outdated lockfile, or a route mismatch after moving a page.

When troubleshooting release docs, always verify which version you are editing. Content in `docs/` updates the current stream, while snapshots in `versioned_docs/version-x.y/` remain frozen unless explicitly edited. This distinction prevents accidental edits to historical release documentation.

## Quick Recovery Checklist

1. Run `npm run build` locally and fix any reported link or markdown errors
2. Confirm `baseUrl` matches the repository path exactly
3. Check that `versions.json` includes the expected release order
4. Verify translated files exist under the correct locale folder
5. Re-run CI after committing fixes to `main`
