# Getting Started

TechDocs is designed for internal product teams who need fast, reliable documentation updates without waiting on an engineering release cycle. Writers can edit markdown directly, push changes, and see updates on the live docs site in the same deployment pipeline used by developers.

For a new writer or engineer, the onboarding flow is intentionally short: clone the repository, install dependencies, and run the local server. The local preview mirrors production navigation, search behavior, and version menus, so review feedback is based on what users will actually see.

Before your first publish, confirm that your GitHub account has write access to the repository and that your branch strategy is clear. Teams that work with release branches should keep day-to-day edits in the current docs and only create a version snapshot when a release candidate is approved.

## Quick Setup Checklist

1. Clone the repo and install dependencies
2. Start local preview with `npm run start`
3. Edit pages in the `docs/` folder
4. Open a pull request for review
5. Merge to `main` to trigger auto-deploy
