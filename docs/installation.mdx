# Installation

This project uses a standard Node-based setup so onboarding is predictable across engineering and documentation teams. The only required local dependencies are Node.js 20+ and npm. No extra CLI tools are needed for day-to-day writing and publishing.

For local setup, install packages once and run the development server to preview markdown changes in real time. This gives authors immediate feedback for navigation, links, search indexing, and page hierarchy before opening a pull request.

Production installation is handled by the GitHub Actions workflow in `.github/workflows/deploy.yml`. Any commit merged into `main` triggers a clean build and deploy to GitHub Pages, which means there is no manual publishing step and no hidden release process outside the repository.

## Recommended Install Flow

```bash
git clone https://github.com/SulagnaSasmal/my-docs-practice.git
cd my-docs-practice
npm install
npm run build
npm run start
```

If `npm run build` passes locally, the same content is ready for CI deployment.
